# AdsPlatformsFinder

Проект AdsPlatformsFinder представляет собой веб-приложение для управления и поиска рекламных платформ на основе их локаций. Приложение позволяет загружать данные о платформах через файл и выполнять поиск по заданным путям.

## Технологии

- .NET (C#)
- ASP.NET Core
- Swagger (для документации API в режиме разработки)

## Структура проекта

### Основные компоненты

1. **Program.cs**  
   Настраивает и запускает веб-приложение, регистрирует сервисы и middleware, включая Swagger для документации API.

2. **Node.cs**  
   Модель данных, представляющая узел дерева для хранения информации о рекламных платформах и их локациях. Содержит методы для обработки и поиска данных.

3. **AdsPlatformsController.cs**  
   Контроллер API, предоставляющий два основных эндпоинта:
   - `upload` — для загрузки данных о рекламных платформах из файла.
   - `load` — для поиска рекламных платформ по заданному пути.

## Как использовать

### Запуск приложения

1. Убедитесь, что у вас установлен .NET SDK.
2. Клонируйте репозиторий.
3. Перейдите в папку проекта и выполните команду:
   ```bash
   dotnet run --urls "http://localhost:5000;https://localhost:5001"
   ```
4. Приложение будет доступно по адресу https://localhost:5001 или http://localhost:5000

## Использование API
### **Загрузка данных (upload)**  
Отправьте POST-запрос на `/api/AdsPlatforms/upload` с файлом, содержащим данные о рекламных платформах.  
Формат файла:

```text
Название платформы:путь1,путь2,...
```
Пример:

```text
Ревдинский рабочий:/ru/svrd/revda,/ru/svrd/pervik
Газета уральских москвичей:/ru/msk,/ru/permobl,/ru/chelobl
```
### **Поиск данных (load)**
Отправьте POST-запрос на `/api/AdsPlatforms/load` с параметром path, содержащим путь для поиска.  
Пример запроса:

```text
/api/AdsPlatforms/load?path=/ru/svrd
```
### Ответ:

```json
["Ревдинский рабочий", "Крутая реклама"]
```
